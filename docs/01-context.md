# 프로젝트 컨텍스트

## 이게 뭔가요
지인(아는 누나)의 결혼을 축하하기 위해 선물로 직접 만들어주는 **모바일 청첩장 웹앱**.

## 만드는 사람
- Spring Boot 백엔드 개발자 (Docker, Nginx, AWS EC2, Jenkins CI/CD 등 인프라 경험 있음)
- 이번 프로젝트는 평소 스택과 반대로, **서버 없는 가벼운 구조**로 진행하기로 결정함
  (트래픽이 예식 전후 몇 주에 몰리고, 사용자 수도 적어서 상시 서버 운영이 불필요하다고 판단)

## 진행 상황
1. 청첩장 샘플 구조 분석 완료 (섹션 구성 파악)
2. 기술 스택 방향 결정: **Next.js + Firebase(Firestore/Storage) + Vercel** (서버 없음)
3. 클래식/우아한 톤의 **정적 HTML 목업 완성** (`wedding-invitation.html`)
   - 디자인 시스템(색상: 아이보리+와인+골드, 타이포: Noto Serif KR + Cormorant Garamond) 확정
   - 실제 신랑/신부 정보, 예식 정보, 오시는 길 정보 반영 완료
   - 스크롤 리빌 애니메이션 + 페이지 진입 애니메이션 적용
   - 카카오톡 공유하기(feed 타입) 연동 코드 작성, JS 키 반영
   - Timeline, Interview 섹션은 불필요 판단하여 제거
4. 이제 이 HTML을 Next.js 프로젝트로 옮기고, Firebase 연동해서 실제 동작하는 서비스로 만드는 단계

## 이번 클로드 코드 세션의 목표
정적 HTML 목업 → Next.js 컴포넌트 기반 실제 서비스로 전환.
방명록, RSVP, 갤러리 업로드 등 실제 데이터가 오가는 기능을 Firebase로 구현.

## 제약 / 참고사항
- **일반적인 모바일 청첩장 장르의 구성**(사진+타임라인+갤러리+방명록+계좌+지도)을 참고해 처음부터 새로 디자인한 것임
- 배포 도메인 미정 — Kakao 공유, OG 메타태그의 URL은 실제 도메인 확정 후 채워야 함
- 실제 계좌번호, 신랑/신부 MBTI·취미 해시태그는 아직 미정 (placeholder 상태)
