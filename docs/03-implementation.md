# 구현 사항 (인프라 세팅 완료 이후)

`wedding-invitation.html`이 완성된 디자인/마크업/CSS 원본. 아래는 이걸 Next.js 컴포넌트로
옮기면서 **실제로 동작하게 만들어야 하는 것들**만 정리 (Firebase 프로젝트 생성 등 인프라 작업 제외).

> 이 문서는 연락하기/계좌/RSVP/화환 모달, Ending 물결 애니메이션이 반영된 최신 HTML 기준으로 업데이트됨.

## 1. 정적 → 컴포넌트 변환 (그대로 옮기기만 하면 되는 것)
- [ ] `<style>` 내 CSS 변수(`:root`)를 `globals.css`로 이동
- [ ] 각 섹션을 `components/sections/*.tsx`로 분리 (구조는 `02-infra.md` 참고)
- [ ] SVG 디바이더를 `<Divider />` 공통 컴포넌트로 추출 (반복 사용되는 요소)
- [ ] 4개 모달(연락하기/RSVP/화환)의 공통 오버레이 마크업을 `<Modal />` 공통 컴포넌트로 추출
      → `.modal-overlay`, `.modal-box`, `.modal-close` 구조 재사용
- [ ] 폰트 로드를 `next/font`로 전환 (Cormorant Garamond, Noto Serif KR) — 현재는 `<link>` 태그 방식

## 2. 애니메이션 로직 (JS → React 훅)
- [ ] 스크롤 리빌: `IntersectionObserver` 로직을 `useScrollReveal` 커스텀 훅으로 전환
      → `.reveal`, `.reveal-group` 클래스 토글 방식 그대로 가져가되, React ref 기반으로 재작성
- [ ] 진입 애니메이션(`load-in` fadeInUp)과 Ending 물결 애니메이션(`waveFlow`)은 순수 CSS라 그대로 유지 가능 — 컴포넌트 이전 시 별도 로직 불필요
- [ ] `prefers-reduced-motion` 대응 유지 (물결 애니메이션 포함, 이미 처리되어 있음)

## 3. 카운트다운 & 캘린더
- [ ] `useCountdown(weddingDate)` 훅으로 분리 — 현재 `setInterval` 로직 그대로 옮기되 컴포넌트 언마운트 시 clear 처리 추가
- [ ] 캘린더는 현재 하드코딩된 마크업 → 날짜 계산 로직으로 자동 생성하도록 개선 (달이 바뀌어도 재사용 가능하게)

- [ ] 비밀번호 대조 후 본인 글 삭제 가능하게

## 5. RSVP (참석 의사 전달) — UI 완성, 저장 로직 미구현
현재 상태: 모달 UI(신랑측/신부측 토글, 참석/불참석 토글, 성함 입력, 식사 여부 토글,
개인정보 동의 체크박스, 조건부 활성화되는 제출 버튼)까지 전부 완성. `submitRsvp()`는
현재 `alert()`만 띄우고 닫히는 placeholder 상태.
- [ ] `submitRsvp()` 내부를 Firestore `rsvp` 컬렉션 `create` 호출로 교체

## 6. 계좌번호 — UI/복사 기능 완성, 실제 데이터만 남음
현재 상태: 신랑측/신부측 그룹 단위 아코디언, 그룹 안에 본인+부모님 3명씩 row로 정리됨.
복사 버튼은 `navigator.clipboard.writeText()`로 실제 동작함 (복사 시 아이콘 색상 피드백 포함).
- [ ] 지금은 더미 계좌번호 → 실제 계좌 정보로 교체 (개발 작업 아님, 콘텐츠 수급 필요)
- [ ] React 전환 시 아코디언 열림 상태를 `useState`로 관리 (`toggleAccount` 로직 전환)
- [ ] (선택) 계좌번호를 코드에 하드코딩하지 않고 Firestore/환경변수로 분리할지 결정
      — 개인정보성 데이터라 git에 커밋되는 게 부담스러우면 고려
